    Comparación de Impacto

    1
    Aislamiento de Fallos (Fault Isolation)

    Si el servicio de alertas por correo se bloquea, el resto del banco (transferencias, consultas de saldo, pagos) sigue operando normalmente sin degradación general.

    2
    Despliegues Independientes (Independent Deployment)

    ¿Quieres subir un cambio visual al catálogo de beneficios? Compilas y redespliegas solo ese microservicio en 1 minuto, sin tener que reiniciar ni testear todo el core transaccional del banco.

    3
    Escalabilidad Granular (Granular Scaling)

    En días de alto tráfico (como el Black Friday), levantas 10 réplicas del servicio de Pagos con Docker en segundos. El servicio de Notificaciones sigue consumiendo el mínimo de recursos.

    4
    Diversidad Tecnológica (Polyglot Stack)

    El núcleo transaccional corre en Java 21 / Spring Boot por su robustez; el recomendador de productos corre en Python por sus librerías de IA, y el chat en Node.js. Cada módulo usa la herramienta ideal.

    Protocolos

    Fundamentos REST: Los Hilos de la Red 📡

    Para que los microservicios cooperen, deben comunicarse mediante un protocolo común. Aprende los verbos y códigos HTTP en este lab en vivo.

    Verbos HTTP (Los Métodos)

    GET
    Leer: Obtiene un recurso sin modificarlo. GET /products/1
    POST
    Crear: Envía datos para crear algo nuevo. POST /orders
    PUT
    Actualizar: Reemplaza completo el recurso. PUT /users/5
    DELETE
    Eliminar: Borra el recurso del servidor. DELETE /users/5
    PATCH
    Modificar parcial: Actualiza solo algunos campos. PATCH /users/5/email

    Códigos de Estado HTTP

    2xx
    Éxito. 200 OK · 201 Created · 204 No Content
    3xx
    Redirección. 301 Moved · 304 Not Modified
    4xx
    Error del cliente. 400 Bad Request · 401 Unauthorized · 403 Forbidden · 404 Not Found
    5xx
    Error del servidor. 500 Internal Server Error · 503 Service Unavailable

    ⚙️ Dominio

    El corazón de la aplicación. Contiene las reglas de negocio puras escritas en Java simple, sin depender de Spring, JPA ni ninguna librería externa.

    🔌 Puerto (Interfaz)

    Un contrato que el Dominio define para comunicarse con el exterior. Siempre es una interface Java sin implementación concreta.

    ✔ interface XxxPort
    ✔ Sin anotaciones Spring
    ✔ Solo firmas de métodos

    Adaptabilidad en Entornos Corporativos 🛡️

    En el desarrollo profesional, los entornos corporativos suelen tener políticas de seguridad avanzadas. Aquí te mostramos cómo configurar tu entorno de forma segura y eficiente.

    1. Configuración de Red para Gestores de Dependencias

    Cuando trabajamos bajo una intranet corporativa, es habitual configurar los proxies de red en herramientas como Maven o Gradle para permitir la correcta descarga de librerías.

    Ubicación sugerida: ~/.m2/settings.xml

    2. Gestión Dinámica de Puertos de Red

    Si un puerto como el `:8080` ya está asignado a otro servicio del sistema local, podemos redirigir la escucha de nuestro microservicio de manera sencilla.

    Configuración: server.port=9000 (en application.properties)
